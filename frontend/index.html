Open this page from the same host as the API (e.g. /ui/). Audits use POST /analyze/async; POST /analyze remains for scripts.

Step 2: Fix code from Git repository

    
After a successful audit, enter the clone URL of the repo to patch. The API clones into .clones/<run-id>, checks out branch test (from origin/test if it exists, otherwise creates test from the default branch), runs code_modify, then npm run build. Set SEO_AGENT_GIT_AUTO_PUSH=1 in .env to run git push -u origin test after a successful local commit (off by default). Requires git on the server PATH and push-capable credentials for the remote.

    
Auto-fix scope: Detects src/app vs app, optional pages/ / src/pages/, maps URLs to route files, then patches missing_meta on the resolved layout.tsx / layout.jsx. duplicate_h1 and missing_alt run on the route file plus a bounded import walk and evidence substring search (so shared components can be updated). broken_link gets a suggested source file when the link text appears in the tree; set SEO_AGENT_FIX_BROKEN_LINKS=1 in .env for optional conservative internal href replacement (off by default).
