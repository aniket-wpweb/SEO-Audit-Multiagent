SEO Agent

Step 1 runs a read-only audit. Step 2 clones a Git repository (HTTPS or git@), checks out the test branch (creates it if missing), applies fixes, runs npm run build when files change, then commits locally on test (no push). Open this page from the same host as the API (e.g. /ui/). Audits use POST /analyze/async; POST /analyze remains for scripts.

Enter the site URL and crawl limits. Large sites can take several minutes; a progress indicator runs while the audit request is in flight.

Step 2: Fix code from Git repository

    
Enter the Git clone URL for the site you audited. The API clones into .clones/<run-id>, checks out branch seo-fixes (or SEO_AGENT_GIT_BRANCH), runs seo_fix (rules + optional OpenAI via SEO_FIX_API_KEY), verifies with npm run build when the repo is Node, then commits locally. Set SEO_AGENT_GIT_AUTO_PUSH=1 to push. Requires git on PATH and push credentials for private remotes.

    
Note: The audit crawls the live URL; fixes apply to the clone. Redeploy from the patched branch for the crawl to reflect changes.
